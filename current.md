# 思考OS構築（このテーマの正本）

## 背景 / なぜ考えているか
- 生成AIとの壁打ちが、その場限りで終わってしまい、思考が積み上がらないという課題がある
- 会話ログを保存すること自体が目的ではなく、
  「前回どこまで考えていて、次に何を考えるべきか」が即座に分かる状態を作りたい
- 思考は本質的に非構造だが、再開のためには構造化された足場が必要という矛盾を解きたい
- 壁打ちの気持ちよさ（テンポ・相棒感）を犠牲にせずに、
  思考の連続性と再開性を成立させるための環境として「思考OS」を構築している

---

## 登り方の全体像（フェーズ地図・仮）
※ フェーズは暫定。今後の運用・違和感に応じて更新される前提。

- **Phase1：成立検証フェーズ（完了）**
  - 非構造対話 → 構造化 → 再開 が成立するかを検証
  - skills 分離（wall_buddy / structurizer / resume_guide）
  - minutes / current の役割分担が機能することを確認

- **Phase2：拡張フェーズ（現在地）**
  - 思考の正本を外部（GitHub）に置く
  - current.md の自動同期（push型）を前提にした再開体験を作る
  - ChatGPTアプリでの壁打ち体験を主軸に固定する

- **Phase3：常用フェーズ（将来）**
  - デバイス・場所を意識せずに思考を再開できる
  - 自動化・ショートカット・API連携を含めた日常利用

---

## 思考の全体像（論点地図）
- 壁打ちは **ChatGPTアプリ**が最も体験が良く、主役に据える
- AIの役割は「思考主体」ではなく、
  - 伴走（wall_buddy）
  - 書記（structurizer）
  - 再開支援（resume_guide）
- 会話ログは正本にしない
- 思考状態を表す **current.md を唯一の正本**とする
- minutes は会話単位の議事録として積み上げる
- current.md は毎回上書きされる「現在の思考状態」
- ChatGPTは外部ドキュメントを自動で pull できない
- そのため **自動同期は push 型で設計する**
  - 「常に最新 current.md が渡された状態で再開できる」体験を目指す
- 同期・保存は ChatGPT ではなく、周辺システム（GitHub / Shortcut / API）が責務を持つ

---

## 現在地
- Phase1 を完了し、Phase2 に入った
- 以下は確定事項：
  - 壁打ちは ChatGPT アプリで行う
  - 構造化（minutes / current）は一区切り時のみ行う
  - current.md / minutes は GitHub を正本置き場とする
- 現在検討・実装中の焦点：
  - current.md を **自動的に同期（push）し**
    次回の壁打ちが常に最新状態から始まる運用
  - 「自動同期しているように感じる再開体験」をどう作るか

---

## 運用ルール（更新版・暫定）
- 対話は非構造で自由に行う（壁打ち）
- 一区切りついたら `structurizer` を呼び、
  - minutes を新規作成
  - current.md を更新
- current.md / minutes は GitHub リポジトリに保存する
- 再開時は、最新の current.md が前提として渡される
  （ChatGPTが自動取得するのではなく、仕組み側で push する）
- 自動同期の責務は ChatGPT ではなく、外部仕組みが担う
